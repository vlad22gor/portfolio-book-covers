# Рекомендация по стеку сайта

Дата: 2026-03-27

## Короткий вывод

Для текущего этапа сайт-портфолио обложек книг строим как статический визуальный сайт.

Рекомендуемый стек:

- `Astro`
- `TypeScript`
- `Tailwind CSS v4`
- нативные `image` + `video/webm/mp4`
- локальные данные в `Markdown`/`JSON`/`TS`
- статический деплой на `GitHub Pages`, `Cloudflare Pages` или `Netlify`

## Почему этот стек

Основная цель сайта сейчас:

- показать обложки максимально эстетично
- сохранить контроль над качеством картинки
- убрать лишнюю инженерную сложность
- держать поддержку и развитие предсказуемыми

Связка `static imagery + short motion loops` оптимальна для этой цели.

## Почему Astro остаётся лучшей основой

- быстрый на статических страницах
- хорошо подходит для SEO
- не требует лишнего клиентского JavaScript
- удобен для ведения кейсов как локального контента

## Рекомендуемая архитектура проекта

### Основа

- `Astro` pages/layouts/components
- `src/content` или `src/data` для кейсов
- `public/assets/cases/...` для изображений
- `public/assets/motion/...` для `webm/mp4/poster`

### Контент кейсов

Поля frontmatter:

- `slug`
- `title`
- `year`
- `project`
- `shortDescription`
- `cover`
- `motion` (опционально)
- `gallery`
- `seo`

### Motion-слой

Каждый кейс при необходимости получает:

- `poster.webp`
- `loop.webm`
- `loop.mp4`

Принцип:

- видео короткое
- постер обязателен
- autoplay только при корректном muted/inline поведении

## Что не рекомендую

- строить UX вокруг тяжёлой интерактивности без явной пользы для презентации
- добавлять клиентский JS без измеримой необходимости

## Практическое решение

1. Главная: статичный hero или постер-кадр с коротким loop рядом или внутри hero.
2. Галерея кейсов: лёгкие 2D-карточки.
3. Страница кейса: сильные статики + один или несколько коротких `webm`/`mp4`.
4. Дальнейшее усложнение делать только под конкретные визуальные задачи.

## Итог

Рекомендованный стек на текущий этап:

`Astro + TypeScript + Tailwind + static images + webm/mp4 + static hosting`
