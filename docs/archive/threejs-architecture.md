# Архив: 3D-подсистема на three.js

Дата: 2026-03-22

## Статус

Этот документ сохранён как архивное описание ранее выбранного направления с `three.js`.

Текущее решение проекта:

- активную `3D viewer`-архитектуру не развиваем
- `three.js`-слой не считаем частью текущего production-пути
- основная подача кейсов строится через `static imagery + webm/mp4 motion`

## Почему документ не удалён

Мы не удаляем его полностью, потому что в нём есть полезный исследовательский контекст:

- прошлые решения по `BookViewer3D`
- ограничения procedural viewer
- структура 3D-данных и ассетов
- наработки, к которым можно вернуться позже

## Когда к нему можно вернуться

Возвращение к 3D имеет смысл только если одновременно выполняются условия:

- есть отдельное время на R&D
- есть готовность строить более серьёзный pipeline через `C4D` или другой DCC
- есть понятная причина, почему motion-loop уже недостаточен
- качество интерактивного результата реально превышает controlled `webm/mp4` presentation

До этого момента документ считается законсервированным.

## Что было в прошлой гипотезе

Ранее 3D-слой планировался как:

- `three.js`
- `react-three-fiber`
- `@react-three/drei`
- procedural viewer книги из front/back/spine ассетов

Сейчас эта гипотеза не является целевой архитектурой проекта.
